# Contributing to Novus

Thanks for contributing.

## Before You Start

1. Read the README and open issues.
2. Comment on an issue before starting significant work.
3. Ask if the issue is unclear.
4. Do not begin large architecture changes without discussion.

## Workflow

Issue
→ assign/comment
→ create branch
→ implement
→ test
→ open pull request
→ review
→ merge

## Branch Names

Use clear names:

- `feat/new-feature`
- `fix/bug-name`
- `docs/update-readme`
- `test/add-tests`

## Pull Requests

A pull request should include:

- what changed
- why it changed
- screenshots for UI changes
- tests added or updated
- known limitations
- linked issue

## Rules

- Do not push directly to `main`.
- Do not commit secrets, API keys, `.env` files, or tokens.
- Keep pull requests focused.
- Ask before adding major dependencies.
- Generated AI output must be validated before use.

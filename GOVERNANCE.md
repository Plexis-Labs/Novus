# Novus Governance

## Purpose

Novus is a student open-source organization built around shared responsibility, visible contribution, and real engineering practice.

GitHub does not provide equity shares for an organization. Fairness is handled through:

- equal governance rights for founding owners
- public contribution history
- pull request authorship
- subsystem ownership
- release notes
- contributor documentation
- written decision rules

## Roles

### Founding Owners

Founding owners are GitHub Organization Owners.

They share responsibility for:

- organization continuity
- core repository protection
- major product direction
- release approval
- team standards
- financial or legal decisions

Founding-owner status is not based only on who clicked “Create organization.”

### Core Maintainers

Core maintainers own a subsystem or recurring responsibility.

Examples:

- frontend
- backend
- AI/ML
- testing and CI
- documentation/design

Core maintainers receive visible ownership through:

- `CODEOWNERS`
- issue ownership
- pull request history
- release notes
- `CONTRIBUTORS.md`

### Contributors

Contributors work through issues and pull requests.

Every meaningful contribution is credited publicly.

## Major Decisions

The following require unanimous approval from all current founding owners:

- deleting, transferring, or renaming the organization
- deleting, transferring, or privatizing core repositories
- changing the project license
- adding paid billing commitments
- monetizing, selling, or transferring project assets
- adding or removing a founding owner

## Technical Decisions

Technical decisions should happen in issues, pull requests, or Architecture Decision Records.

Subsystem maintainers propose decisions for their area.

If there is a deadlock, founders discuss it. The goal is consensus, not hierarchy.

## Credit

Credit follows meaningful contribution.

Credit is recorded through:

- commits
- pull requests
- code review
- release notes
- `CONTRIBUTORS.md`
- subsystem ownership
- project documentation

Equal governance does not mean pretending everyone contributed equally to every part.

## Promotion

A contributor may become a core maintainer after either:

- three meaningful merged pull requests, or
- ownership of a shipped subsystem,

and consistent participation during a milestone.

Organization Owner access is earned through sustained responsibility and trust. It is not automatically granted to every new member.

## Conduct

Be direct, respectful, and specific.

Raise blockers early. Review code seriously. Do not claim work you did not do.
